# AstrBot 监督与内容播报插件

这个插件提供四类能力：

1. 给自己创建监督任务。
2. 在群聊里 `@别人` 给对方创建监督任务。
3. 目标用户发言时，按提醒间隔触发催促提醒。
4. 每天固定时间发送“更新内容”和“内容预告”。

提醒文案支持优先调用当前会话正在使用的大模型，并按当前 AstrBot 人格或自定义人格要求生成；如果模型不可用，会自动回退到本地模板。

## 可复制指令清单

### 监督

```text
监督 开始 任务=写周报 待办=整理数据、写摘要 时长=2h 冷却=1h 抽取=2
监督 开始 @小明 任务=做海报 待办=出图、排版 时长=2h 冷却=1h
监督 状态
监督 状态 @小明
监督 完成
监督 完成 @小明
监督 取消
监督 取消 @小明
监督 帮助
```

旧别名 `督工` 仍然可用。

这些命令现在支持两种触发方式：

1. 直接发送，例如 `监督 开始 ...`
2. `@机器人` 后再发，例如 `@机器人 监督 开始 ...`

参数说明：

1. `任务=` 必填。
2. `待办=` 可选，多个待办可用 `、`、`，` 或换行分隔。
3. `时长=` 可选，例如 `30m`、`2h`、`1d`。
4. `冷却=` 可选，就是提醒间隔，例如 `10m`、`2h`。
5. `抽取=` 可选，每次提醒展示几条待办。

### 更新内容

```text
更新内容 设置 时间=21:00 内容=今天更新了第一章和封面
更新内容 开
更新内容 关
更新内容 状态
更新内容 立即发送
更新内容 立即发送 今天先发一条测试
```

### 内容预告

```text
内容预告 设置 时间=20:00 内容=明天预告第二章和设定图
内容预告 开
内容预告 关
内容预告 状态
内容预告 立即发送
内容预告 立即发送 明天晚上放预告图
```

## 设置页

插件设置里现在有三块常用配置：

1. `设置页监督任务`
2. `设置页内容播报`
3. `正常对话让路前缀`

### 设置页监督任务

命令创建的进行中任务会同步显示到这里；任务完成、取消或失效后会自动从这里移除。

建议这样理解字段：

1. `开始时间`：留空就是立即开始。
2. `持续时间 / 结束时间`：二填一；两个都填时，以结束时间为准。
3. `期间提醒间隔`：目标发言后，至少隔多久才再提醒一次。
4. `提醒次数`：填 `0` 表示不限制。
5. `持续时间` 支持普通时长，也支持 `每天`、`永久`。

私聊任务最少填写这些：

```text
目标用户ID=QQ号
任务=写第一章
开始时间=2026-04-16 20:00
待办=大纲、正文、校对
持续时间=3h
期间提醒间隔=2h
提醒次数=0
```

群聊任务还需要填写 `群号`，或者直接写完整会话 ID：

```text
napcat:GroupMessage:群号
```

### 设置页内容播报

命令设置过的“更新内容”和“内容预告”也会同步显示到这里；从这里删掉后，对应播报会一起取消。

建议这样理解字段：

1. `发送到哪个会话`：填完整会话 ID，例如 `napcat:FriendMessage:123456` 或 `napcat:GroupMessage:654321`。
2. `会话备注`：只是方便自己看，不影响发送。
3. `每天几点发`：按 `HH:MM` 填。
4. `文案`：到点后原样发出。
5. `启用`：关闭后保留设置，但不会实际发送。

### 正常对话让路前缀

这里是一条一条添加的前缀列表，例如：

```text
/
／
！
```

如果目标消息以这些前缀开头，监督插件会让给 AstrBot 正常对话或命令处理，不会截走消息。`@机器人` 的让路固定保留，不依赖这个列表。

## 大模型人格催促

如果开启了 `llm_enabled`：

1. 插件会优先使用当前会话对应的聊天模型生成催促文案。
2. 如果开启了 `llm_follow_active_persona`，会把当前 AstrBot 会话人格一起带入。
3. 还可以用 `llm_custom_prompt` 追加专门的监督语气要求。

例如：

```text
说话像盯稿的编辑，语气凶一点，短句，不卖萌，但不要辱骂。
```

## 已知限制

1. 当前监督提醒仍然是“见到人再催”，不会自己单独轮询骚扰。
2. 私聊或自监督任务会跟着目标用户走；群聊里 `@别人` 创建的任务仍然只在原群里触发。
3. 如果目标消息是在显式唤醒 AstrBot，例如 `@机器人` 或 `/chat`，监督插件会让路，不抢占正常对话和命令处理。
4. 同一个用户同一时间只允许存在一个进行中的监督任务。
5. “更新内容”和“内容预告”当前是“每日固定时间 + 固定文案”的实现。
