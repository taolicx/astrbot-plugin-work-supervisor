# AstrBot 监督与更新预告插件

这个插件提供四类能力：

1. 给自己创建监督任务。
2. 在群聊里 `@别人` 给对方创建监督任务。
3. 目标用户发言时，按提醒间隔触发催促提醒。
4. 每日固定时间发送“更新内容”和“内容预告”。

本插件的提醒文案支持优先调用当前会话使用中的大模型，并按当前 AstrBot 人格或自定义人格要求生成催促话术；如果模型不可用，会自动回退到本地模板。

## 监督命令

```text
监督 开始 任务=写第一章 待办=大纲、正文、校对 时长=3h 冷却=2h 抽取=3
监督 开始 @小明 任务=做海报 待办=出图、排版 时长=2h 冷却=1h
监督 状态 [@目标]
监督 完成 [@目标]
监督 取消 [@目标]
监督 帮助
```

旧版 `督工` 命令仍作为兼容别名可用。

参数说明：

1. `任务=` 必填。
2. `待办=` 可选，用顿号、逗号或换行分隔。
3. `时长=` 可选，例如 `30m`、`2h`、`1d`。
4. `冷却=` 可选，例如 `10m`、`2h`，冷却内不会重复催。
5. `抽取=` 可选，每次提醒展示几条待办。

## 播报命令

```text
更新内容 设置 时间=21:00 内容=今天更新了第一章和封面
更新内容 开
更新内容 关
更新内容 状态
更新内容 立即发送

内容预告 设置 时间=20:00 内容=明天预告第二章和设定图
内容预告 开
内容预告 关
内容预告 状态
内容预告 立即发送
```

## 设置页添加任务

插件配置里有 `设置页监督任务` 列表，可以直接新增任务。命令创建的进行中任务也会同步显示到这里；任务完成、取消或失效后会自动从列表移除。

设置页里建议按这几个字段理解：

1. `开始时间`：留空就是立刻开始。
2. `持续时间 / 结束时间`：二填一。两个都填时，以结束时间为准。
3. `期间提醒间隔`：目标发言后，至少隔多久才再提醒一次。
4. `提醒次数`：填 `0` 表示不限制。
5. `持续时间` 支持 `30m`、`2h`、`1d`，也支持 `每天`、`永久`。

私聊任务最少填写：

```text
目标用户ID=QQ号
任务=写第一章
开始时间=2026-04-16 20:00
待办=大纲、正文、校对
持续时间=3h
期间提醒间隔=2h
提醒次数=0
```

群聊任务还需要填写 `群号`，或直接填写完整 `会话ID`：

```text
napcat:GroupMessage:群号
```

如果不填写完整会话ID，插件会按 `平台ID`、`会话类型`、`目标用户ID` 或 `群号` 自动生成。默认平台ID是 `napcat`。

时间字段说明：

1. `开始时间` 留空时，表示从你保存配置的当前时刻开始。
2. 如果填了 `结束时间`，就按结束时间结束。
3. 如果只填 `持续时间`，插件会按 `开始时间 + 持续时间` 自动计算结束时间。
4. 如果 `持续时间=永久`，任务不会自动结束。
5. 如果 `持续时间=每天`，任务会按开始时间每天循环。
6. 如果 `开始时间` 还没到，机器人不会提前催。

## 大模型人格催促

如果启用了 `llm_enabled`：

1. 插件会优先使用当前会话对应的聊天模型生成催促话术。
2. 如果开启了 `llm_follow_active_persona`，会把当前 AstrBot 会话人格一并带入。
3. 还可以用 `llm_custom_prompt` 追加专门的监督语气要求。

例如可以把 `llm_custom_prompt` 配成：

```text
说话像盯稿的编辑，语气凶一点，短句，不卖萌，不要太客气，但不要辱骂。
```

## 已知限制

1. 当前版本的监督提醒仍然是“见到人再催”，不会自己单独轮询骚扰。
2. 私聊或自监督任务会跟着目标用户走，目标用户在别的会话里发言时也会触发提醒；群聊里 `@别人` 创建的任务仍然只在原群里触发。
3. 如果目标消息是在显式唤醒 AstrBot，例如 `@机器人` 对话或 `/chat` 这类前缀命令，监督插件会让路，不抢占正常对话和命令处理。
3. 一个用户同一时间只允许存在一个进行中的监督任务。
4. “更新内容”和“内容预告”当前是“每日固定时间 + 固定文案”的实现。
